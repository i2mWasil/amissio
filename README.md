Amissio is an advanced, textbook-quality study note generator that transforms YouTube videos into comprehensive, structured knowledge. By combining audio transcriptions, visual frame analysis, Optical Character Recognition (OCR), and a Map-Reduce hierarchical summarization pipeline powered by Google's Gemini models, Amissio ensures no critical information is lost—whether spoken, written on slides, or shown in code snippets.

---

## 💡 The Idea
Video tutorials, lectures, and conference talks contain a wealth of information, but extracting structured, readable notes from them is incredibly tedious. Traditional transcription tools miss the visual context (like slides, diagrams, and code snippets). 

Amissio solves this by using a **Multimodal Retrieval-Augmented Generation (RAG)** approach. It "watches" and "listens" to the video, extracts text and visual context, chunks it semantically, and synthesizes perfect study notes that you can also chat with.

## ✨ Key Features
- **YouTube Video Processing**: Automatically downloads and processes audio and video streams using `yt-dlp`.
- **Multimodal Extraction**:
  - **Audio**: High-quality, timestamped transcriptions via `faster-whisper`.
  - **Vision & OCR**: Extracts key frames based on image hashing thresholds to avoid duplicates. Uses vision models and OCR to pull text and context from slides or whiteboard content.
- **Semantic Chunking**: Intelligently chunks transcript and visual data based on semantic meaning and token limits.
- **Vector Database**: Uses Qdrant to store and retrieve high-dimensional embeddings generated by Gemini.
- **Map-Reduce Summarization**: Hierarchically generates textbook-quality notes by summarizing segments and merging them, bypassing context-window limits for long videos.
- **Interactive Chat Panel**: Chat directly with the generated notes or the video's knowledge base using the built-in RAG API.
- **Terminal-Style UI**: A sleek, developer-focused React frontend built with Vite and Tailwind CSS v4.

## 🛠️ Tech Stack

### Backend
- **Framework**: Python 3 / FastAPI
- **LLM / AI**: Google Gemini API (`gemini-2.5-flash`, `text-embedding-004`)
- **Vector DB**: Qdrant
- **Audio Processing**: faster-whisper
- **Video Extraction**: yt-dlp, OpenCV
- **Image Processing**: Pillow, ImageHash

### Frontend
- **Framework**: React 19 / Vite
- **Styling**: Tailwind CSS v4
- **Icons**: lucide-react
- **Rendering**: react-markdown (for Markdown notes)

## 🏗️ Architecture & How It Works
1. **Ingestion**: User submits a YouTube URL via the Terminal UI.
2. **Extraction**:
   - `yt-dlp` fetches the best audio and video streams.
   - `faster-whisper` creates a timestamped transcript.
   - OpenCV extracts frames at a set FPS; image hashing deduplicates visually similar frames (e.g., unchanged presentation slides).
3. **Processing**:
   - Vision models process keyframes to extract OCR text and describe visual context.
   - The combined transcript and visual data is semantically chunked.
4. **Embedding**: Chunks are embedded using `text-embedding-004` and stored in **Qdrant**.
5. **Synthesis**: A Map-Reduce chain prompts `gemini-2.5-flash` to generate detailed, structured notes from the chunks.
6. **Interaction**: The frontend displays the markdown notes and allows RAG-based chatting with the vector database for Q&A.

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- Python 3.10+
- [Qdrant](https://qdrant.tech/) instance (running locally via Docker or Qdrant Cloud)
- Google Gemini API Key

### 1. Backend Setup
1. Navigate to the backend directory:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows use `.venv\Scripts\activate`
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Configure Environment Variables:
   Copy the example config and fill in your Gemini API key and Qdrant details.
   ```bash
   cp .env.example .env
   ```
5. Start the backend server:
   ```bash
   fastapi dev main.py
   # Or using uvicorn: uvicorn main:app --reload
   ```
   The API will be available at `http://localhost:8000`.

### 2. Frontend Setup
1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the development server:
   ```bash
   npm run dev
   ```
   The frontend will be available at `http://localhost:5173`.

## ⚙️ Configuration (`.env`)
The backend requires several environment variables to function correctly. Key variables include:

| Variable | Description | Default |
|----------|-------------|---------|
| `GEMINI_API_KEY` | Your Google Gemini API Key | - |
| `GEMINI_MODEL` | Gemini model for summarization/chat | `gemini-2.5-flash` |
| `GEMINI_VISION_MODEL` | Gemini model for image/OCR analysis | `gemini-2.5-flash` |
| `GEMINI_EMBEDDING_MODEL`| Model for vector embeddings | `text-embedding-004` |
| `QDRANT_URL` | URL to your Qdrant instance | `http://localhost:6333` |
| `WHISPER_MODEL_SIZE` | Size of the faster-whisper model | `base` |

*(See `backend/.env.example` for the full list of configurable parameters including chunking logic and retry resilience.)*
